# AIDD Agent Platform — 后端分阶段实施计划 (Backend Implementation Plan)

> 本文档将《后端设计文档》中的技术架构拆解为可落地的、分布式的实施步骤。由于前端设计仍在讨论中，本计划纯粹聚焦于后端（API、数据库、对象存储、LangGraph 核心）的代码实现路径。

---

## 阶段 1：基础设施与骨架搭建 (Infrastructure & Scaffolding)
**目标**：拉起项目结构，配置好所有依赖的云原生中间件，确保环境变量与模型正确连接。

1. **环境初始化**：
   - 使用 `mamba` 创建 `environment.yml`，锁定 FastAPI、SQLAlchemy 2.0、LangGraph、`google-genai` 等核心依赖。
2. **中间件启动 (本地调试模式)**：
   - 编写本地开发的 `docker-compose.yml`（包含 PostgreSQL 16、Redis 7、SeaweedFS）。
   - 初始化 SeaweedFS S3 Bucket（如 `aidd-data`）。
3. **配置管理层实现**：
   - 实现 `core/config.py`，使用 `pydantic-settings` 统一读取 DB、Redis、S3、Gemini 的环境变量。
4. **数据库与 ORM 初始化**：
   - 建立 `app/models/`，实现 `User` 和 `Session` 的 SQLAlchemy 声明式模型。
   - 配置 Alembic，运行第一次 DB Migration，生成用户和会话的底层表。

---

## 阶段 2：混合存储双轨制与会话 API (Hybrid Storage & Session APIs)
**目标**：彻底告别臃肿的 Postgres 聊天记录表，打通 Redis 热缓存与 SeaweedFS 冷存储。

1. **认证与基础用户路由**：
   - 实现基于 `python-jose` 的简单 JWT 登录与拦截器。
2. **SeaweedFS 异步客户端**：
   - 使用 `aiobotocore` 封装 S3 工具类（支持上传 JSONL 消息流、追加文件、生成预签名 URL）。
3. **混合存储读写器 (Storage Manager)**：
   - **写**：封装函数，拦截 LangGraph 返回的消息。短文本入 Redis 刷新过期时间，长日志旁路写入 SeaweedFS。
   - **读**：封装函数，当 Redis 缓存击穿时，去 SeaweedFS 读取 `messages.jsonl` 重建历史状态。
4. **会话 CRUD API 实现**：
   - 完成 `/sessions` 的增删改查 REST 接口。

---

## 阶段 3：精简工具层与降维管道 (Lean Tools & Preprocessing Pipeline)
**目标**：复用 Biomni 资产，但加入防爆显存的硬核“脱水”处理。

1. **核心工具移植与硬修剪 (Hard Pruning)**：
   - 从 Biomni 提取 `query_pubmed` 和 `query_chembl` 的底层请求逻辑。
   - 强制套用 Pydantic Model，抛弃无关元数据，严格限制返回的 JSON 结构。
2. **工具搜索与动态挂载 (Deferred Tools)**：
   - 实现 `ToolSearchTool`，将不常用的化学分析计算工具放入动态库，支持模型需要时“热重载”。
3. **Map-Reduce 预处理引擎**：
   - 针对批量文献返回，编写使用低成本模型（如 Gemini 2.5 Flash）的并发摘要脚本，确保工具返回给主 Agent 的数据**永远不超过 3000 Tokens**。

---

## 4. 阶段 4：Prompt 分层引擎与 LangGraph 核心编排 (Agent Core)
**目标**：使主 Agent 拥有身份、规则，并能流式地与工具进行交互。

1. **4层 Prompt 渲染器**：
   - 利用 Jinja2 编写 `System Prompt` 模板，动态注入系统时间、当前可用工具清单。
   - 实现 `Assistant Pre-fill` 强制在模型输出末尾追加 `<thought>` 约束格式。
2. **LangGraph 状态图定义**：
   - 构建 `AgentState`。
   - 实现 `AgentNode`（负责思考和发号施令）与 `ToolNode`（负责执行代码）。
   - 配置条件路由（Conditional Edges），打通 ReAct 推理闭环。
3. **防幻觉与 Gemini Grounding 接入**：
   - 对接 `google-genai` SDK，开启 Google Search Grounding。
   - 在后处理逻辑中拦截 `grounding_metadata`，为模型生成的回答强制打上 `[1]` 引用上标。

---

## 阶段 5：高阶调度、子代理与自动压缩 (Advanced Management)
**目标**：解决“复杂问题界面卡死”和“聊久了模型失忆”两个核心痛点。

1. **实现 Agent-Tool Duality (子代理机制)**：
   - 开发 `AgentTool`。拦截该工具调用，在后台 `asyncio.create_task` 启动新的 LangGraph 实例（Subagent）。
   - 将 Subagent 的详细推理过程打包存入 SeaweedFS（Sidechain Transcript），仅返回精简结果。
2. **Auto-Compaction (会话自动压缩)**：
   - 实现动态阈值计算逻辑（根据模型 Max_Length 动态触发）。
   - 编写摘要压缩逻辑，删除早期 Message 列表，将摘要填入 System Prompt 的 `<memory_context>` 槽位。
   - 引入 Circuit Breaker 熔断器，防止 API 错误导致的死循环压缩。

---

## 阶段 6：WebSocket 推送与 L3 追踪对接 (Streaming & Observability)
**目标**：让前端能够优雅地展示所有的推理过程和进度动画。

1. **WebSocket 端点建立**：
   - 建立 `/ws/chat/{session_id}` 路由。
2. **全链路流式推送协议**：
   - **Token 流**：将 LLM 吐出的文字流式推给前端。
   - **Event 流**：子代理后台查数据库时，推送 `{"type": "subagent_status", "msg": "正在阅读第 5 篇文献..."}`。
   - **Trace 流**：当主模型调用工具时，下发精简版的 ToolCall 参数供前端画出手风琴折叠面板。
3. **后端综合联调与 API 冻结**：
   - 利用 Postman 或编写 Pytest 异步测试用例，从“创建用户”到“通过 WS 完成一轮包含工具调用的长对话”，跑通全链路。
   - 输出正式版 OpenAPI 文档，交接给前端团队。
