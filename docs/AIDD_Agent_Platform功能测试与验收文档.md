# AIDD Agent Platform — 功能测试与验收文档

> 本文档基于前期的架构设计（产品、前端、后端设计文档），将所有技术特性转化为可验证的**功能点、测试用例与验收标准 (Acceptance Criteria, AC)**。作为 QA 与最终产品验收的基准。

---

## 1. 基础架构与会话管理 (Session & Auth)

### 1.1 用户认证与隔离
- **功能描述**：系统提供轻量级的多用户隔离环境，确保用户的对话与敏感查库记录相互隔离。
- **测试用例**：
  - **TC-1.1.1**：用户 A 注册并登录，创建一个名为 `靶点 EGFR 研究` 的对话。
  - **TC-1.1.2**：用户 B 登录，系统不应显示用户 A 的 `靶点 EGFR 研究` 对话。
  - **TC-1.1.3**：通过非法的 JWT Token 请求 `/api/v1/sessions`，应返回 `401 Unauthorized`。
- **验收标准 (AC)**：
  - [ ] 密码使用 bcrypt 哈希加密存储。
  - [ ] 跨用户越权访问 API 会被立刻拒绝（HTTP 401/403）。

### 1.2 会话 CRUD 与流式输出
- **功能描述**：支持多轮自然语言对话，并在前端实现 ChatGPT 样式的字级别流式打印。
- **测试用例**：
  - **TC-1.2.1**：发送消息 “你好”，UI 立即展示用户的消息卡片，随后 AI 回复开始平滑流式输出。
  - **TC-1.2.2**：在一个长时间未操作的闲置对话中重新发送消息，Redis 缓存穿透后能从 SeaweedFS 成功拉取历史记录恢复对话。
- **验收标准 (AC)**：
  - [ ] WebSocket 连接建立后的 TTFT (Time To First Token) 延迟低于 1000ms。
  - [ ] 会话列表支持新建、删除、重命名。

## 2. 领域知识检索与工具预处理 (Tooling & Data Pipeline)

### 2.1 动态工具挂载 (Deferred Tools)
- **功能描述**：平台默认隐藏 24 个长尾数据库工具，仅通过 `ToolSearchTool` 动态加载，防止 Token 爆炸。
- **测试用例**：
  - **TC-2.1.1**：用户提问“请查阅最新的心血管文献”，模型只调用默认的 `query_pubmed`。
  - **TC-2.1.2**：用户提问“请在化合物数据库中对比这些分子的活性”，模型先调用 `ToolSearchTool("化合物活性")`，随后在下一轮上下文中成功识别并调用被热重载的 `query_chembl` 工具。
- **验收标准 (AC)**：
  - [ ] 默认状态下，发给大模型的 System Prompt 中包含的工具 Schema 少于 5 个。
  - [ ] 动态挂载的工具能被模型立刻识别并正确填写 JSON 参数。

### 2.2 大规模搜索结果脱水管道 (Preprocessing)
- **功能描述**：防止几万字原始文献把大模型冲刷到“失忆”，必须经过修剪、重排和 Map-Reduce。
- **测试用例**：
  - **TC-2.2.1 (硬修剪)**：查询一个包含上百个无用元数据字段的 UniProt 蛋白质，Agent 收到的内部 JSON 只有 `Sequence`, `Function`, `Keywords`。
  - **TC-2.2.2 (重排)**：提交一篇 5 万字的英文文献文本，大模型最终只收到并阅读了通过 Embedding 筛选出的最相关的那 1000 个字。
- **验收标准 (AC)**：
  - [ ] 任何单一工具返回给大模型的 Payload 尺寸**绝对不可超过 3000 Tokens**。

## 3. Agent 高阶调度与子代理 (Subagent Scheduling)

### 3.1 子代理即工具 (Agent-Tool Duality)
- **功能描述**：复杂的科研综述任务不应阻塞主对话流，主 Agent 应能启动一个封闭运行的后台子代理（Subagent）。
- **测试用例**：
  - **TC-3.1.1**：用户下达宏大指令“深度调研 T790M 突变的所有抑制剂及临床阶段”。
  - **TC-3.1.2**：主 Agent 决定调用 `AgentTool(background=True)`。前端立刻显示状态 Pill `[Deep Research Agent 正在思考...]`。
  - **TC-3.1.3**：子代理在后台查询了 15 次不同的数据库并报错重试 2 次。完成后，主 Agent 醒来并给出了一份精华的 Markdown 总结报告。
- **验收标准 (AC)**：
  - [ ] 主 Agent 的上下文 `messages` 中**不会**包含子代理那 15 次琐碎的内部查库细节。
  - [ ] 前端必须能通过 WebSocket 实时接收子代理内部执行状态的非阻塞通知。

### 3.2 探索与规划模式 (Plan Mode)
- **功能描述**：高危/重型计算前，强制 Agent 先规划。
- **测试用例**：
  - **TC-3.2.1**：模型自动挂载 `EnterPlanModeTool`，随后输出了一份 Markdown 格式的行动计划（Plan）。
- **验收标准 (AC)**：
  - [ ] 在 Plan Mode 下，系统会硬拦截针对数据库的写操作或消耗极大的分子对接计算工具。

## 4. 上下文管理与混合存储 (Context & Storage)

### 4.1 Auto-Compaction (会话自动压缩)
- **功能描述**：在达到动态上下文阈值时，自动将早期聊天记录压缩为精华摘要并存入 System Prompt。
- **测试用例**：
  - **TC-4.1.1**：持续与模型进行长对话（模拟灌入 80k tokens）。
  - **TC-4.1.2**：系统触发 `should_auto_compact`，将前 50 条历史记录替换为一段 300 字的 `<memory_context>` 摘要。模型依然能准确回答第 1 轮设定的“核心靶点是什么”。
- **验收标准 (AC)**：
  - [ ] 连续 3 次压缩失败则触发熔断器（Circuit Breaker），防止无意义的 API 调用浪费。

### 4.2 双轨制物理存储 (Dual-Track Storage)
- **功能描述**：精简数据喂给模型，海量原始数据旁路直写 SeaweedFS。
- **测试用例**：
  - **TC-4.2.1**：工具返回了几兆的 PubMed XML 原文。
  - **TC-4.2.2**：在 SeaweedFS 的 `traces/raw_outputs/` 目录中可以找到该 XML 文件。
- **验收标准 (AC)**：
  - [ ] 写入 PostgreSQL 的 `Session` 表仅有不到 1KB 的元数据。所有几十兆的原始 Log 都在 SeaweedFS 独立存放。

## 5. 防幻觉与溯源机制 (Anti-Hallucination)

### 5.1 强制引用与段落级溯源
- **功能描述**：大模型的每一句科学论断都必须有可验证的出处。
- **测试用例**：
  - **TC-5.1.1**：用户提问“目前最火的减肥药靶点机制是什么”。
  - **TC-5.1.2**：模型回复的机制解析中，每段话结尾都带有 `[1]` 或 `(PMID: 3233...)` 形式的引文标记。
  - **TC-5.1.3**：点击 `[1]`，前端根据后端的 `raw_data_uri` 解析出一个 SeaweedFS 预签名 URL，展示大模型看过的原始数据。
- **验收标准 (AC)**：
  - [ ] 通过 Gemini Grounding 或 PubMed 工具返回的内容，必须在最终答案中强绑定。
  - [ ] 如果搜索结果中没有相关事实，大模型必须回答“目前检索未发现相关资料”，严禁强行编造分子式或文献。

## 6. Agent 追踪审计面板 (Tracing UI)

### 6.1 Level 3 全链路可观测性
- **功能描述**：专业开发与研究人员可以随时展开面板，审阅 AI 每一步究竟在想什么。
- **测试用例**：
  - **TC-6.1.1**：打开前端的 Trace 面板，可以看到一个手风琴展开组件。
  - **TC-6.1.2**：点击 `Step 2: Act`，面板能高亮显示 Agent 当时向 `query_chembl` 发送的确切 JSON 参数，以及工具返回的截断版 JSON 结果。
  - **TC-6.1.3**：界面底部的 Status Bar 会显示本次对话累计消耗了 `24,500 Tokens`，折合 `$0.03`。
- **验收标准 (AC)**：
  - [ ] 所有被“压缩丢弃”的对话节点，也能在 Trace 面板的 `Sidechain Transcript` 里找到完整留存。
  - [ ] 消耗金额与延迟毫秒数 (Latency) 显示准确。
